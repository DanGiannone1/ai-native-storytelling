# AI Foundations: The Narrative

## Overview

**Title:** AI in 2026: From Chatbots to Autonomous Agents
**Audience:** Business executives, technologists, anyone seeking AI literacy
**Duration:** ~25 minutes (19 slides)
**Tone:** Educational, aspirational, grounded in reality

---

## The Story We're Telling

This presentation takes the audience on a journey from "AI is just chatbots" to understanding how autonomous agents are fundamentally changing how businesses operate. We start where they are (familiar with chatbots) and end where we want them to be (seeing autonomous agents as the future of business).

---

## Part 1: Foundation (Slides 1-6)

> "AI in 2026: From chatbots to autonomous agents"

---

### Slide 2: AI History Timeline
**Template:** `TimelineRiver`
**Message:** AI didn't start with chatbots. It's been quietly powering products for decades.

**Key points:**
- 1950: Turing Test (the question was asked)
- 1956: Dartmouth (AI as a field was born)
- 1997: Spam filters (AI in your inbox)
- 2006: Recommendations (personalization at scale)
- 2011: Voice assistants (mobile assistants)
- 2016: Smart feeds (AI deciding what you see)
- 2022: Conversational AI (the moment it surfaced)

**Punchline:** "Quietly powering products - until it surfaced"

---

### Slide 3: The Conversational AI Moment
**Template:** `Highlight`
**Message:** November 2022 was a watershed moment. AI became conversational, accessible, undeniable.

### Slide 4: What is an LLM?
**Template:** `Explainer`
**Message:** Demystify the technology. An LLM is a pattern-matching engine trained on human knowledge.

## Part 2: The Limitation & The Solution (Slides 7-9)

### Slide 7: The Limitation
**Template:** `Barrier`
**Message:** This is the pivot point. LLMs are brilliant - but they can't DO anything.

**The contrast:**
| Can Generate | Can't Execute |
|--------------|---------------|
| Write your email | Send it |
| Write code | Execute it |
| Plan your meeting | Book it |

**Punchline:** "A brilliant advisor with no hands"

---

### Slide 8: Tools Bridge the Gap
**Template:** `RadialTools`
**Message:** Tools are what give LLMs the ability to act. Each tool is a new capability.

**Visual:** Tools arranged radially around central LLM - each tool is a capability (API, Database, Email, Calendar, etc.)

---

### Slide 9: The AI Agent Formula
**Template:** `AgentLoop`
**Message:** This is the key insight. An agent is simply: LLM + Tools + Loop.

**The formula:**
```
LLM + Tools + Loop = Agent
```

**The loop:** Think → Act → Observe → Reflect → (repeat)

**Punchline:** "A loop that runs until the task is done"

## Part 3: Agents at Work (Slides 10-13)

### Slide 10: Agents in Action
**Template:** `TaskCards`
**Message:** Show concrete examples. Agents don't just chat - they complete entire workflows.

**Three examples:**
1. **Process expense reports:** Scan receipts → Extract amounts → Categorize → Submit
2. **Fix a bug:** Read logs → Find cause → Write fix → Test → Commit
3. **Monitor systems:** Check metrics → Detect anomalies → Alert → Report

**Punchline:** "Multi-step work - planned and executed end to end"

## Key Themes Throughout

1. **Demystification:** AI isn't magic. It's pattern matching + tools + loops.
2. **Progression:** Chat → Tools → Agents → Multi-Agent → Self-Improving
3. **Reality:** We're not pitching future tech. We're showing what runs today.
4. **Possibility:** The audience should leave thinking "this changes everything"

## Notes for Presenters

- **Pace:** Let the animations breathe. Don't rush transitions.
- **Emphasis:** Slide 7 (Limitations) and Slide 9 (Agent Formula) are the pivot points.
- **Grounding:** Always connect back to "this is running today in production"
- **Close strong:** Slide 19 should feel like an arrival, not a pitch
